# AI Fitness Coach

> Персональный ИИ-тренер, который создаёт тренировочные программы под ваши
> цели и уровень подготовки.

Современное веб‑приложение (React + TypeScript + TailwindCSS + Material UI на
Next.js) с адаптивным интерфейсом для мобильных устройств и ПК, светлой и
тёмной темами. Работает **без обязательной регистрации и без базы данных на
MVP‑этапе** — все данные пользователя сохраняются локально в браузере через
LocalStorage.

---

## Возможности

| Раздел | Что делает |
| --- | --- |
| **Главная** | Hero + кнопка «Создать программу», обзор возможностей. |
| **Анкета** | Возраст, пол, рост, вес, уровень, цель, частота, место, оборудование, травмы, заметки. |
| **Программа** | Сгенерированный план по дням с изображениями упражнений + экспорт **TXT / PDF**. |
| **Анализ тренировки** | Фактические веса/повторения + самочувствие → рекомендация: увеличить / уменьшить / оставить вес. |
| **AI‑Тренер** | Чат, адаптирующий программу под ситуацию (травма, усталость, нехватка времени, только гантели…). |
| **Прогресс** | Графики Chart.js: изменение веса тела и рабочих весов по упражнениям. |
| **Библиотека** | Каталог упражнений с описанием, мышцами и изображением; поиск + фильтр. |

---

## Технологический стек

- **Frontend:** React 18, TypeScript, Next.js 14 (Pages Router), TailwindCSS, Material UI v6.
- **Backend:** Next.js API Routes.
- **AI:** оркестратор моделей с провайдерами **Groq / OpenAI / Gemini** и встроенным **Mock**‑провайдером (работает офлайн, без ключей).
- **Графики:** Chart.js + react-chartjs-2.
- **Хранение:** LocalStorage (см. раздел «Архитектура» — слой спроектирован под замену).

## Быстрый старт

```bash
# 1. Установка зависимостей
npm install

# 2. Запуск в режиме разработки (http://localhost:3000)
npm run dev

# 3. Продакшн-сборка
npm run build && npm start
```

**Приложение полностью работает «из коробки» без API‑ключей** — оркестратор
автоматически использует Mock‑провайдер, чтобы можно было демонстрировать
продукт локально.

### Подключение настоящей LLM

1. Скопируйте `.env.example` → `.env.local`.
2. Задайте ключ нужного провайдера и выберите его в `LLM_PROVIDER`:

```bash
LLM_PROVIDER=groq          # или openai | gemini | mock
GROQ_API_KEY=...
# OPENAI_API_KEY=...
# GEMINI_API_KEY=...
```

3. Перезапустите `npm run dev`. Имя провайдера, ответившего на запрос,
   отображается в шапке программы (чип «Источник»).

## Архитектура

Ключевая идея — **зависимость от интерфейсов, а не реализаций**, поэтому
MVP можно расширять, не трогая UI и бизнес‑логику.

### 1. Хранилище (Repository pattern)

```
React-хуки (useStorage)  →  DataRepository (интерфейс)
                              ├── LocalStorageRepository   ← сейчас
                              ├── SupabaseRepository       ← будущее
                              └── PostgresRepository       ← будущее
```

Реализована только `LocalStorageRepository`. Интерфейс `DataRepository`
спроектирован так, чтобы завтра добавить `SupabaseRepository` /
`PostgresRepository` одной новой реализацией — экраны переписывать не нужно
(они зависят только от хуков).

### 2. LLM-оркестратор

```
Сервисы (program/analysis/chat)  →  ModelOrchestrator  →  LLMProvider
                                      (выбор + fallback)    ├── Groq
                                                            ├── OpenAI
                                                            ├── Gemini
                                                            └── Mock (всегда)
```

- `ModelOrchestrator` выбирает провайдера по `LLM_PROVIDER`, при сбое
  перебирает остальных, и гарантированно заканчивает `Mock`.
- Добавить нового провайдера = реализовать интерфейс `LLMProvider` и
  зарегистрировать его в `buildProviders()`.
- Системный промпт тренера (из ТЗ) живёт в `lib/llm/prompts.ts`.

### 3. API-роуты — тонкий транспорт

Все роуты (`/api/generate-program`, `/api/analyze-workout`, `/api/chat`)
обёрнуты в `withApi`, который централизованно делает: гейт по HTTP‑методу,
try/catch с единым форматом ошибок, CORS‑заголовки. Вся «интеллектуальная»
часть — в сервисах.

## Подготовлено под будущую архитектуру

Эти возможности **не реализованы**, но архитектурно под них оставлены
очевидные точки расширения:

| Будущее | Точка расширения |
| --- | --- |
| Supabase / PostgreSQL | `DataRepository` — новая реализация + выбор в `getRepository()`. Env-ключи уже в `.env.example`. |
| Авторизация | `withApi` — место для проверки JWT/session перед handler. |
| Платная подписка / Stripe | отдельный сервис биллинга + middleware перед генерацией программы. Ключи в `.env.example`. |
| Telegram Bot | переиспользование `services/*` + webhook‑роут; CORS уже разрешён. |

## Генерация изображений упражнений

Иллюстрации — минималистичные SVG‑плейсхолдеры (Notion/Apple Fitness vibe),
генерируются из `src/data/exercises.json`:

```bash
node scripts/generate-exercise-images.js
```

Чтобы заменить на реальные фотографии — положите файлы в
`public/images/exercises/` (или обновите поле `image` в JSON).
